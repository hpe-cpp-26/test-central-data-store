Comprehensive Guide to Data Cleaning in Machine Learning
Data cleaning—often referred to as data cleansing or preprocessing—is one of the most critical steps in the machine learning workflow. Raw data is notoriously messy, filled with missing values, duplicates, inconsistencies, and outliers. Standard machine learning models follow the "Garbage In, Garbage Out" principle; feeding them low-quality data will inevitably result in poor predictions and unreliable insights.
Properly cleaned data ensures that algorithms can identify genuine underlying patterns without being misled by noise.
1. Handling Missing Data
Missing values are a common occurrence due to human error, system failures, or data omissions during collection. Leaving missing values unaddressed can cause algorithms to fail or produce heavily biased results.
Identification
Check for explicit null markers like NaN, Null, or None.
Look out for implicit indicators used during data entry, such as 999, -1, ?, or empty strings "".
Strategies for Resolution
Deletion:
Listwise Deletion: Remove the entire row/record. Best used when the missing values are completely random and affect less than 5% of the dataset.
Pairwise Deletion: Drop specific columns if a feature is missing a massive percentage of its data (e.g., >60%) and holds little predictive value.
Imputation:
Statistical Imputation: Replace missing numerical values with the Mean (for normally distributed data) or Median (robust against outliers). Use the Mode for categorical values.
Predictive Imputation: Use algorithms like $K$-Nearest Neighbors ($KNN$) or iterative regression to predict and fill the missing values based on other available features.
Constant Value: Fill with a flag like "Unknown" or 0 to explicitly preserve the information that the data was missing.
2. Managing Outliers
Outliers are data points that deviate significantly from the rest of the observations. While some outliers represent genuine anomalies, others are the product of measurement errors or data corruption.
Detection Techniques
Z-Score: Measures how many standard deviations a data point is from the mean. Typically, a $|Z| > 3$ indicates an outlier.
Interquartile Range (IQR): Calculates the range between the 25th percentile ($Q_1$) and the 75th percentile ($Q_3$).
$$\text{IQR} = Q_3 - Q_1$$
Data points falling outside the bounds $[Q_1 - 1.5 \times \text{IQR}, Q_3 + 1.5 \times \text{IQR}]$ are flagged as outliers.
Strategies for Resolution
Trimming: Remove the outlier rows entirely if they are confirmed errors.
Winsorization (Capping): Cap the extreme values at a designated percentile (e.g., resetting values above the 99th percentile down to the 99th percentile value).
Transformation: Apply logarithmic or square root transformations to reduce the skewness caused by extreme values.
3. Removing Duplicates and Irrelevant Data
Duplicate and irrelevant observations artificially bloat the dataset and skew model weights, causing the model to over-index on repetitive information.
Identical Duplicates: Remove rows where every single feature is a perfect match with another row.
Partial Duplicates: Identify records that share unique identifiers (like User_ID or Transaction_ID) but have slight variations in other columns due to system lag or double-logging. Keep the most recent or accurate record.
Irrelevant Features: Drop columns that have zero variance (e.g., a column where every row is 1) or features that hold no structural relationship to the target variable (e.g., random tracking URLs or internal database UUIDs).
4. Resolving Structural Inconsistencies
Structural errors happen when data is collected from multiple sources or entered manually, leading to naming variations that confuse categorical models.
Typographical Errors & Naming Conventions: Standardize text variations. For example, ensure that "N.Y.", "ny", and "New York" are all mapped to a single standard string: "New York".
Inconsistent Capitalization: Convert all string features to lowercase to prevent the model from treating "Admin" and "admin" as two distinct categories.
Unit Standardization: Ensure all measurements use the same scale. Convert weights entirely to kilograms, currencies to a single base denomination, or distances to kilometers.
5. Type Conversion and Formatting
Machine learning models require precise data types to run optimization mathematics efficiently.
Data Type Enforcement: Ensure numerical columns containing numbers are not accidentally stored as object/string types. Convert boolean flags (e.g., "Yes"/"No") into explicit 1 and 0 integers.
Date and Time Standardisation: Convert date strings into proper DateTime objects. Standardize all time stamps to a unified timezone (like UTC) to accurately calculate durations, intervals, or cyclical time features.
