# Netflix System Architecture: Troubleshooting Guide

This document outlines common failure states, diagnostic strategies, and resolution steps for a global, high-availability video streaming platform.

## 1\. Video Streaming & Playback (Open Connect CDN)

The video delivery network is responsible for the actual streaming of large video files. Problems here directly impact the user's viewing experience.

**Symptoms:** Users experience heavy buffering, degraded video resolution, or "Title Not Available" playback errors.

**Diagnosis:** The client application is failing to connect to the nearest Open Connect Appliance (OCA), or the specific video asset is missing from the regional cache.

**Resolution:**

1.  Verify client DNS routing to ensure requests are resolving to the optimal OCA within the user's local ISP.
    
2.  Check the AWS Control Plane logs to confirm the requested title was successfully proactively cached to the edge servers during off-peak hours.
    
3.  Initiate a traffic failover to route the user's request to a neighboring OCA cluster or a centralized AWS fallback server.

## 2\. API Gateway & Microservice Routing

The API Gateway handles all incoming traffic from user devices and routes it to hundreds of backend microservices.

**Symptoms:** The application loads endlessly, user profiles fail to appear, or the system returns HTTP 500 errors upon login.

**Diagnosis:** The API Gateway is struggling to route requests, or a critical downstream microservice (e.g., the User Auth service) has become unresponsive.

**Resolution:**

1.  Review the circuit breaker metrics dashboard to see if a downstream service's failure threshold has been crossed (tripped open).
    
2.  Confirm that fallback mechanisms are successfully serving stale cached data or a degraded default UI instead of crashing the client application.
    
3.  Analyze distributed tracing logs to isolate the exact microservice causing the latency bottleneck and restart the affected pods.
    

* * *

## 3\. Data Consistency & State Management

Netflix relies on distributed NoSQL databases (like Cassandra) and high-performance caching layers (like EVCache) for high availability across global regions.

**Symptoms:** Viewing history fails to update, newly added list items disappear, or a recently created user profile vanishes.

**Diagnosis:** The system is experiencing replication lag across database nodes, or there is a localized failure in the caching layer resulting in cache misses.

**Resolution:**

1.  Monitor the Cassandra cluster for read/write latencies, dropped mutations, or node synchronization issues.
    
2.  Check the cache hit/miss ratio on the caching layer; if a node is down, monitor the automated self-healing process as it repopulates from the primary database.
    
3.  Validate that cross-region replication is functioning so that user state remains consistent even if a region experiences partial degradation.
